Core Features

Built for paranoids.
Designed for everyone.

Every component of Anchor Cloud is engineered with security-first architecture. No backdoors. No metadata harvesting. No compromise.

True Zero-Knowledge Architecture
Files are encrypted client-side before they ever leave your device. We never see your data — mathematically impossible to access.
# AES-256 EAX mode encryption pipeline
cipher = AES.new(user_key, AES.MODE_EAX)
ciphertext, tag = cipher.encrypt_and_digest(file_data)
# Nonce + tag + ciphertext → stored. Key → never stored.

Process

How It Works

A four-step pipeline that takes your file from raw bytes to encrypted vault entry — all in milliseconds.

01
You Select a File
Drop any file into the vault. JavaScript collects file metadata and raw binary data from your browser.
02
Sent to Python API
A FormData POST request sends the file to the FastAPI backend over HTTPS. No plaintext touches the disk.
03
Encrypted + UUID Assigned
Python generates a UUID v4 for the file, encrypts it with AES-256 EAX, and stores it as a "message" in the vault engine.
04
Logged to MySQL
File metadata (UUID, name, size, type, owner, timestamp) is committed to the MySQL database. The encrypted blob is stored separately.
